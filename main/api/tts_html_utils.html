tts_html_utils.resource.compile_scripts(scripts)

Extracts JavaScript content and wraps it in HTML <script> tags.

This function goes one step further than compile_styles. Since JavaScript cannot just be dumped into the page (it must be inside a <script> block), this function reads the content and then uses the script_general.html template to wrap it properly.

Parameters:

scripts (list) – A list of JS file paths or raw JS strings.

Returns:

A list of HTML strings, each containing a full <script>…</script> block.

Return type:

list[str]

tts_html_utils.resource.render_html_from_template(template_dir, template_name, **template_kwargs)

The core rendering engine.

This function uses Jinja2 to take a template file (HTML with holes in it) and a dict of python variables (the data to fill the holes) and produces a final HTML string.

Security Note (XSS): We enable autoescape=True for HTML and XML files. This prevents Cross-Site Scripting attacks. If a user tries to inject a variable containing <script>alert(‘hack’)</script>, Jinja automatically converts it to safe text (&lt;script&gt;…) so it displays as text rather than executing as code.

Parameters:

  • template_dir (str or Path) – The directory where the template file resides.

  • template_name (str) – The name of the specific file to load.

  • template_kwargs – Key-value pairs representing variables to inject into the template.

Returns:

The fully rendered HTML string.

Return type:

str
